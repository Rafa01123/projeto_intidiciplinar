Guia Rápido: Git & GitHub

O que é Git?

Git é tipo uma máquina do tempo pro seu código. Ele guarda tudo que você altera e te deixa voltar no tempo se algo der ruim.

  • Salva cada mudança feita.
  • Ajuda times a trabalharem juntos sem bagunça.
  • Permite voltar pra versões antigas.

O que é GitHub?

Pense no GitHub como uma "nuvem" onde você guarda seus projetos do Git e ainda pode colaborar com a galera.

  • Compartilha seu código com o mundo.
  • Permite colaboração (tipo redes sociais pra código).
  • Organiza tarefas, bugs e histórico.

Diferença entre Git e GitHub

  • Git: Programa que roda na sua máquina pra salvar o histórico do projeto.
  • GitHub: Site onde você publica o que fez com o Git.
  • Dá pra usar Git sem GitHub, mas não o contrário.

Comandos básicos do Git Bash

  • ls – Mostra os arquivos na pasta.
  • cd nome_da_pasta – Entra em uma pasta.
  • mkdir nome – Cria uma pasta nova.
  • touch arquivo.txt – Cria um arquivo vazio.
  • clear – Limpa o terminal.

README.md – Exemplo com comandos

git init             # Inicia o Git no projeto
git add .            # Adiciona tudo
git commit -m "msg"  # Salva com uma mensagem
git push origin main # Envia pro GitHub

Como criar um repositório no GitHub

  1. Vá em github.com e faça login.
  2. Clique em "New repository".
  3. Dê um nome e escolha se será público ou privado.
  4. (Opcional) Marque para adicionar README.
  5. Clique em "Create repository".

Clonar um repositório

No terminal, digite:

git clone https://github.com/seu_usuario/seu_repositorio.git

Isso copia o projeto do GitHub pro seu PC.

Como fazer um commit

Depois de mudar arquivos, use:

git add .
git commit -m "Explique o que fez"

Pronto, isso salva uma versão do projeto.

Como enviar pro GitHub (push)

Depois do commit, envie pro GitHub com:

git push origin main

Agora seu código está online.
